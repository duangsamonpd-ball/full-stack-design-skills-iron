# AI Tells Catalog

The recurring giveaways that a UI was generated by default — each with *why* it reads as
AI and a concrete fix. Loaded on demand by the `anti-ai-design-patterns` skill.

How to use: find the tell, apply the fix as a **decision** (a chosen value tied to a token),
not as another layer of decoration.

---

## 1. Color

### 1.1 The default indigo→violet gradient
**Tell:** hero and primary buttons use `#6366f1 → #8b5cf6` (Tailwind `indigo-500`→`violet-500`).
This exact ramp is the single most common AI/SaaS-template signature.

```css
/* ❌ tell */
background: linear-gradient(135deg, #6366f1, #8b5cf6);

/* ✅ decision: commit to one brand hue; use a flat fill or a subtle same-family shift */
background: var(--color-brand);                 /* flat, confident */
/* or a restrained gradient off the obvious hue */
background: linear-gradient(160deg, #0f766e, #115e59);  /* teal, chosen */
```

### 1.2 Everything equally saturated
**Tell:** 6 vivid colors at similar saturation, all competing. Reads as a palette generator output.
**Fix:** one dominant brand color, neutrals doing 80% of the work, accents used sparingly for emphasis only.

### 1.3 Semantic colors as decoration
**Tell:** green/amber/red used because they "look nice" (a green card that means nothing).
**Fix:** reserve success/warning/danger for real states; decorate with brand/neutral instead.

### 1.4 Pure black / pure gray text
**Tell:** `#000` on `#fff`, or the flat `#808080` mid-gray.
**Fix:** slightly tinted near-black (`#111827`) and a real neutral ramp; tie text color to a token.

## 2. Layout

### 2.1 Everything centered
**Tell:** every section is `text-align: center`, `mx-auto`, max-width narrow column all the way down.
**Fix:** left-align text-heavy content (easier to scan); center only deliberate moments (a single hero line, an empty state). Vary alignment to create rhythm.

### 2.2 Endless equal 3-column card grid
**Tell:** feature cards, all identical size/weight, in a perfect 3×N grid, repeated for every section.

```html
<!-- ❌ tell: 6 interchangeable equal cards -->
<div class="grid grid-cols-3 gap-4"> …6 identical cards… </div>

<!-- ✅ decision: a focal item + supporting items (bento / asymmetric) -->
<div class="grid grid-cols-1 gap-md md:grid-cols-3 md:grid-rows-2">
  <article class="md:col-span-2 md:row-span-2">…primary, larger…</article>
  <article>…</article>
  <article>…</article>
</div>
```

### 2.3 Perfect symmetry, no focal point
**Tell:** the eye has nowhere to land; every block is balanced against a twin.
**Fix:** deliberate asymmetry, one clear focal point per screen, active whitespace.

### 2.4 The generic hero
**Tell:** centered headline + one-line subhead + two pill buttons (filled + outline) + faint gradient blob.
**Fix:** give the hero a real idea — a product visual, an asymmetric split, a distinctive type treatment. Kill the decorative background blob.

## 3. Typography

### 3.1 No hierarchy
**Tell:** headings, subheads and body all within a few px; nothing dominates.
**Fix:** a real scale with contrast — the primary element is unmistakably largest.

```
❌ h1 24px · h2 20px · body 16px   (flat)
✅ display 48–64px · h2 28px · body 16–18px · caption 13px   (decisive jumps)
```

### 3.2 Default leading and tracking
**Tell:** everything at browser-default line-height, no letter-spacing intent.
**Fix:** tighten large display type (`leading-tight`, slight negative tracking); open up body (`leading-relaxed`); keep line length ~45–75ch.

### 3.3 Unopinionated system font everywhere
**Tell:** stock `system-ui` / Inter with zero pairing or personality.
**Fix:** choose a typeface with character; pair a display face with a clean text face on purpose. If staying with a system stack, make the *scale and rhythm* the personality.

---

## 4. Icons

### 4.1 An icon on every heading / feature
**Tell:** every card has a little rounded-square icon tile up top.
**Fix:** use icons only where they aid scanning; a strong headline often needs none. (See `icon-library-guide.md`.)

### 4.2 One ubiquitous set used literally everywhere
**Tell:** default line-icon set at default weight, unmodified, on every surface.
**Fix:** pick a set that fits the brand; keep one coherent style; consider light customization.

### 4.3 Decorative icons with no meaning
**Tell:** icons that don't map to any action or concept, added for "visual interest."
**Fix:** drop them, or make them functional; decorative ones get `aria-hidden="true"`.

---

## 5. Copy

### 5.1 Emoji-prefixed headings
**Tell:** `## 🚀 Fast`, `## ✨ Beautiful`, `## 🔒 Secure`.
**Fix:** remove the emoji; let type hierarchy carry emphasis. Reserve emoji for genuinely informal contexts.

### 5.2 Filler value-props
**Tell:** "Elevate your workflow", "Unlock seamless productivity", "Supercharge your team".
**Fix:** say the specific, concrete thing. "Export to CSV in one click" beats "Seamlessly unlock your data."

```
❌ "Elevate your workflow with our seamless, intuitive platform!"
✅ "Turn a Figma frame into a React component in under a minute."
```

### 5.3 Exclamation inflation & hedging
**Tell:** exclamation marks everywhere; or vague hedged claims ("designed to help you potentially improve…").
**Fix:** near-zero exclamation marks; make definite, specific statements.

---

## 6. Spacing

### 6.1 Uniform padding on everything
**Tell:** every section and card uses the same `p-4` / `gap-4`; no density variation.
**Fix:** vary density with intent — group related content tighter, separate unrelated sections more. Use a spacing scale, not one value.

### 6.2 No vertical rhythm
**Tell:** section gaps feel arbitrary; nothing lines up to a system.
**Fix:** adopt a spacing scale (`xs sm md lg xl 2xl`) and stick to it; establish consistent section rhythm.

---

## Priority order when fixing

1. **Color** (the default gradient is the loudest tell) →
2. **Layout focal point** (kill the equal-grid monotony) →
3. **Type hierarchy** →
4. **Copy** →
5. **Icons** →
6. **Spacing rhythm**.

Fixing 1–3 alone removes most of the "this looks AI" impression.
